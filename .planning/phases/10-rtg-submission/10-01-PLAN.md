<task type="auto">
  <name>Task 2: Run secrets scan across codebase and git history</name>
  <files></files>
  <action>
Run grep-based secrets scan to verify no sensitive data will be exposed when the repo goes public. This is a read-only audit task -- no files are created or modified.

1. Scan current files for common secret patterns:
```bash
grep -rn --include="*.ts" --include="*.tsx" --include="*.json" --include="*.toml" --include="*.rs" \
  -E '(PRIVATE_KEY|SECRET_KEY|API_KEY|private_key|secret_key|api_key|-----BEGIN)' .
```
Expected: No matches (or only false positives like type names).

2. Scan git history for leaked secrets:
```bash
git log -p --all -S 'PRIVATE_KEY' --diff-filter=A
git log -p --all -S 'BEGIN PRIVATE' --diff-filter=A
git log -p --all -S 'secret' --diff-filter=A -- '*.env' '*.json' '*.toml'
```
Expected: No matches.

3. Check for .env files that might have been committed:
```bash
git log --all --diff-filter=A -- '*.env' '.env*'
```
Expected: No matches.

4. Verify Anchor.toml wallet reference is a path, not an inline key:
```bash
grep -n 'wallet' Anchor.toml
```
Expected: `wallet = "~/.config/solana/id.json"` (path reference, safe).

If any actual secrets are found, report them and remove/rotate before proceeding. If only false positives (e.g., variable names containing "key" in non-secret context), document and move on.
  </action>
  <verify>
    <automated>echo "Secrets scan is a manual audit -- verify no actionable findings in task output" && grep -rn --include="*.ts" --include="*.tsx" --include="*.json" --include="*.toml" --include="*.rs" -E '(PRIVATE_KEY=|SECRET_KEY=|API_KEY=|-----BEGIN (RSA|EC|DSA|OPENSSH) PRIVATE)' . 2>/dev/null; echo "EXIT:$?"</automated>
  </verify>
  <done>Secrets scan complete with no actionable findings. Anchor.toml wallet is a path reference. No .env files in history. Repository is safe for public release.</done>
</task>
