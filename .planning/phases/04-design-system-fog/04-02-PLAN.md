**1. Create `app/src/lib/mock-data.ts`:**

Define the `MockMarket` interface and export an array of 8-10 diverse mock markets:

```typescript
export interface MockMarket {
  id: string
  question: string
  description: string
  category: 'Politics' | 'Crypto' | 'Sports' | 'Culture' | 'Economics'
  deadline: Date
  resolutionSource: string
  sentiment: 'Leaning Yes' | 'Even' | 'Leaning No'
  poolTotal: string       // formatted string, e.g. "24,800 USDC"
  betCount: number
  status: 'live' | 'resolved'
  outcome?: 'yes' | 'no'  // only if status === 'resolved'
}
```

Include a good variety:
- At least one market per category (Politics, Crypto, Sports, Culture, Economics)
- At least 2 resolved markets (one 'yes' outcome, one 'no' outcome)
- Mix of sentiment values (Leaning Yes, Even, Leaning No)
- Deadlines spread across future dates (use dates relative to now so they always show valid countdowns)
- Realistic prediction market questions (crypto prices, elections, sports outcomes, cultural events)
- Pool totals ranging from small (~500 USDC) to large (~50,000 USDC)
- Bet counts ranging from small (~12) to large (~300+)

Export as `export const MOCK_MARKETS: MockMarket[]`.

**2. Create `app/src/components/market/CountdownTimer.tsx`:**

Simple countdown display component:
```typescript
interface CountdownTimerProps {
  deadline: Date
  className?: string
}
```

Implementation:
- Use React `useState` + `useEffect` with `setInterval` (1-second tick) to compute remaining time.
- Display format: "Xd Xh" if >24h remaining, "Xh Xm" if <24h, "Xm Xs" if <1h, "Ended" if past deadline.
- Clean up interval on unmount.
- Style: `text-xs text-muted-foreground` by default, `text-destructive-foreground` when <1h remaining (urgency indicator).
- Use `cn()` for class merging with the `className` prop.
- Keep it simple -- ~30 lines of component code. No external library needed.

**Layout (top to bottom within card):**
1. **Header row** (flex, items-center, justify-between):
   - Category badge: `text-xs font-medium px-2 py-0.5 rounded-full bg-sage/10 text-sage` (e.g. "Crypto")
   - CountdownTimer showing deadline
2. **Question text**: `text-sm font-semibold text-foreground line-clamp-2 mb-4` (2-line max with ellipsis)
3. **Fogged sentiment** (FogOverlay with `density="light"`, `revealed={isResolved}`):
   - Content: sentiment text, e.g. "Leaning Yes"
   - Style: `text-sm text-muted-foreground`
   - When resolved: shows actual sentiment value, fog cleared
4. **Fogged pool total** (FogOverlay with `density="heavy"`, `revealed={isResolved}`):
   - Content: pool total amount, e.g. "24,800 USDC"
   - Style: `text-lg font-bold text-gold` (gold accent for money amounts)
   - When resolved: shows actual pool total, fog cleared
5. **Stats row** (flex, items-center, gap-4):
   - Bet count: `text-xs text-muted-foreground`, e.g. "142 bets"
   - If resolved: outcome badge (green "Yes won" or red "No won") using `text-xs font-medium px-2 py-0.5 rounded-full`. Green: `bg-emerald/10 text-emerald`. Red: `bg-destructive/10 text-destructive-foreground`.
6. **Quick-bet buttons** (grid grid-cols-2 gap-2, only shown for live markets):
   - Yes button: use Button component with custom class `bg-gold text-gold-foreground hover:bg-gold/90`
   - No button: use Button component with `variant="secondary"`
   - Both `size="sm"`

**Card container:**
- Use CVA for card variants (live vs resolved):
  - Base: `rounded-xl border bg-card p-5 transition-colors cursor-pointer`
  - Live: `border-border hover:border-sage/50`
  - Resolved: `border-border/50 opacity-90` (slightly dimmed to indicate finality)
