  <name>Task 2: Create FogOverlay component with density prop and reveal animation</name>
  <files>app/src/components/fog/FogOverlay.tsx</files>
  <action>
Create `app/src/components/fog/FogOverlay.tsx` -- a composable fog overlay component that layers a frosted glass effect over content.

**Component API:**
```typescript
interface FogOverlayProps {
  density: 'heavy' | 'light'
  revealed?: boolean
  children: React.ReactNode
  className?: string
}
```

**Implementation details:**
1. Outer `div` with `relative overflow-hidden` and the passed `className`.
2. **Content layer** (children): Wrapped in a div with transition on opacity.
   - When `revealed=false`: `opacity-0` (content hidden behind fog).
   - When `revealed=true`: `opacity-100` with `duration-700 ease-out`.
3. **Fog overlay layer**: Absolutely positioned `inset-0` div with:
   - Heavy density: `backdrop-blur-md bg-fog-heavy` (12px blur, 85% opacity)
   - Light density: `backdrop-blur-sm bg-fog-light` (4px blur, 55% opacity)
   - When `revealed=true`: `opacity-0 backdrop-blur-none pointer-events-none` (fog fades out)
   - Transition: `transition-all duration-700 ease-out` (matches reveal timing)
4. **Fog drift inner layer**: Inside the fog overlay, an `absolute inset-0` div with:
   - Gradient: `bg-gradient-to-br from-emerald/20 via-transparent to-sage/15` (subtle emerald-sage gradient)
   - Background-size `200% 200%` to enable the drift animation
   - Animation: `motion-safe:animate-fog-drift` (only plays when user allows motion)
   - When `revealed=true`: drift animation removed (no animation on invisible element)

**Performance notes (from research):**
- Do NOT animate `filter: blur()` directly per-frame -- animate opacity instead while blur remains static. The fog overlay fades out via opacity (GPU-composited) rather than blur value transition.
- Use `backdrop-filter` as a static property on the fog layer. The reveal works by fading the entire fog layer out via opacity, not by reducing blur from Npx to 0px.

**Accessibility:**
- Wrap drift animation in `motion-safe:` Tailwind variant (respects `prefers-reduced-motion`).
- Fog overlay does NOT add any ARIA attributes -- it is purely decorative.

Use `cn()` from `#/lib/utils` for class composition. Import pattern: `import { cn } from '#/lib/utils'`.

Export as named export: `export function FogOverlay({ ... }: FogOverlayProps) { ... }`
