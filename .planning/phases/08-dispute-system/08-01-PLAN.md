**1. register_resolver.rs:**
- Handler takes `amount: u64` parameter (initial stake).
- Validates `amount >= 500_000_000` (500 USDC in lamports).
- Creates `Resolver` PDA via `init`.
- Creates resolver vault token account: seeds `[b"resolver_vault", resolver_wallet.key()]`, authority = Resolver PDA.
- Transfers `amount` USDC from user token account to resolver vault.
- Sets `Resolver.wallet`, `staked_amount = amount`, `approved = false`, `active_disputes = 0`, etc.
- Accounts struct: `resolver_wallet: Signer`, `resolver: Account<Resolver>` (init), `resolver_vault: Account<TokenAccount>` (init), `user_token_account: Account<TokenAccount>`, `usdc_mint: Account<Mint>`, `config: Account<Config>` (for usdc_mint validation), `system_program`, `token_program`, `rent`.

**2. approve_resolver.rs:**
- Only admin (from Config) can call.
- Sets `resolver.approved = true`.
- Adds resolver wallet pubkey to `ResolverRegistry.resolvers` Vec.
- Validates registry not full (`resolvers.len() < MAX_RESOLVERS`).
- Validates resolver not already approved.
- If ResolverRegistry does not exist yet, use `init_if_needed` to create it (seeds `[b"resolver_registry"]`).

**3. stake_resolver.rs:**
- Allows approved resolver to add more USDC stake (top-up).
- Takes `amount: u64` parameter.
- Transfers USDC from user to resolver vault.
- Increments `resolver.staked_amount`.

**4. withdraw_resolver.rs:**
- Two-phase withdrawal: request (sets `withdrawal_requested_at` + `withdrawal_amount`) then execute (after 7-day cooldown = 604800 seconds).
- Takes `amount: u64` and `execute: bool` parameters.
- If `execute == false`: Sets withdrawal request fields. Validates `active_disputes == 0`. Validates remaining stake >= 500 USDC minimum OR amount == staked_amount (full withdrawal).
- If `execute == true`: Validates cooldown elapsed (`clock >= withdrawal_requested_at + 604800`). Validates `active_disputes == 0` again. Transfers from resolver vault to user. Decrements `staked_amount`. Clears request fields.
- If full withdrawal (staked_amount becomes 0): set `approved = false`, remove from ResolverRegistry.

**5. Update resolve_market.rs:**
- Add grace period check: `let grace_deadline = market.resolution_time.checked_add(172800).unwrap();` (48h = 172800 seconds).
- After validating `clock >= market.resolution_time`, add: `require!(clock.unix_timestamp <= grace_deadline, AvenirError::GracePeriodExpired)`.
- This enforces that the creator can only resolve within the 48h window after deadline. After that, the market must go through dispute.

**6. errors.rs** -- Add new error variants:
- `StakeTooLow` ("Resolver stake must be at least 500 USDC")
- `ResolverNotApproved` ("Resolver is not approved")
- `ResolverAlreadyApproved` ("Resolver is already approved")
- `RegistryFull` ("Resolver registry is full")
- `ActiveDisputeExists` ("Cannot withdraw while active in a dispute")
- `CooldownNotElapsed` ("7-day withdrawal cooldown has not elapsed")
- `WithdrawalNotRequested` ("No pending withdrawal request")
- `InsufficientStake` ("Withdrawal would leave stake below minimum")
- `GracePeriodExpired` ("48-hour grace period has expired; market must resolve via dispute")

**7. mod.rs** -- Add the 4 new module declarations and re-exports.
