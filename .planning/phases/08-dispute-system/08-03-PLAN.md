Purpose: This is the core privacy mechanism of the dispute system -- juror votes are encrypted via Arcium MPC so no juror can see any other juror's vote during the voting period. The add_dispute_vote circuit accumulates stake-weighted votes into an encrypted tally.

**add_dispute_vote circuit:**
Takes an encrypted vote (Shared) + existing encrypted tally (Mxe) + plaintext stake weight. Accumulates the stake-weighted vote into the tally.

```rust
/// Accumulate a stake-weighted encrypted vote into the dispute tally.
///
/// Takes:
/// - vote_ctxt: Encrypted bool (is_yes) from juror's shared secret
/// - tally_ctxt: Existing encrypted VoteTotals from MXE
/// - stake_weight: Plaintext u64 (juror's staked amount, passed as plaintext since it's public on Resolver account)
///
/// Returns updated encrypted tally (Mxe-owned). No revealed values -- tally stays private.
#[instruction]
pub fn add_dispute_vote(
    vote_ctxt: Enc<Shared, VoteInput>,
    tally_ctxt: Enc<Mxe, VoteTotals>,
    stake_weight: u64,
) -> Enc<Mxe, VoteTotals> {
    let vote = vote_ctxt.to_arcis();
    let mut tally = tally_ctxt.to_arcis();

    if vote.is_yes {
        tally[0] += stake_weight;
    } else {
        tally[1] += stake_weight;
    }

    tally_ctxt.owner.from_arcis(tally)
}
```

Note: `stake_weight` is a plaintext u64 parameter. The ArgBuilder supports `plaintext_u64` for passing public data into circuits. The stake is public on the Resolver account anyway, so no privacy loss.

**7. cast_vote.rs (the main juror-facing instruction):**

This is the juror's entry point (like place_bet is the bettor's entry point):

Handler takes: `computation_offset: u64`, `vote_ciphertext: [u8; 32]`, `pub_key: [u8; 32]`, `nonce: u128`.

Steps:
1. Validate dispute status == 0 (Voting).
2. Validate voting window: `clock.unix_timestamp <= dispute.voting_end`.
3. Validate caller is a selected juror: find `juror.key()` in `dispute.jurors` and get the juror's index.
4. Validate juror hasn't already voted: `(dispute.votes_submitted >> juror_index) & 1 == 0`.
5. Check MPC lock with 60s timeout recovery (mirror place_bet pattern, but on `dispute.mpc_lock`).
6. Set MPC lock on dispute.
7. Mark juror as having voted: `dispute.votes_submitted |= 1 << juror_index`.
8. Increment `dispute.vote_count`.
9. Read stake weight from Resolver account: `resolver.staked_amount`.
10. Build ArgBuilder (as shown above) and queue add_dispute_vote MPC.

Accounts struct: `#[queue_computation_accounts("add_dispute_vote", juror)]`
- `juror: Signer` (mut)
- `dispute: Account<Dispute>` (mut, PDA seeds `[b"dispute", market_id.to_le_bytes()]`)
- `dispute_tally: Account<DisputeTally>` (PDA seeds `[b"dispute_tally", market_id.to_le_bytes()]`)
- `market: Account<Market>` (for market_id validation, state == Disputed)
- `resolver: Account<Resolver>` (PDA seeds `[b"resolver", juror.key()]`, for stake weight)
- Standard Arcium accounts (mxe, sign_pda, mempool, execpool, computation, comp_def, cluster, pool, clock)
- `system_program`, `arcium_program`
