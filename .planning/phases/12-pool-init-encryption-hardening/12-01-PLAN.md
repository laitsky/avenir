**useCreateMarket.ts** -- Create a hook that chains create_market + init_pool as two sequential transactions. Model on useOpenDispute.ts multi-TX chaining pattern.

Key implementation details:

1. **Step type:** `CreateMarketStep = 'idle' | 'creating' | 'initializing' | 'confirming' | 'retrying' | 'success' | 'error'`

2. **Partial success tracking:** Use `marketCreated` state flag (boolean) and `newMarketId` state (number | null). On retry, skip create_market if `marketCreated` is true and go straight to init_pool.

3. **Mutation flow:**
   - Read Config PDA to determine next market ID: `const [configPda] = getConfigPda(); const config = await program.account.config.fetch(configPda); const nextMarketId = (config as any).marketCounter + 1`
   - Send create_market TX with all required parameters (question, description, category, resolutionSource, resolutionTime, minBet). The mutation takes these as input parameters.
   - Wait for confirmation (`'confirmed'`), set `marketCreated = true` and `newMarketId = nextMarketId`
   - Send init_pool TX using the same Arcium account derivation pattern from useOpenDispute (dynamic import of @arcium-hq/client, derive 12 accounts). Use `getCompDefAccOffset('init_pool')` for the comp_def_account. The init_pool instruction only takes `computation_offset: u64` as argument.
   - init_pool accounts: `{ payer: publicKey, marketPool: getMarketPoolPda(nextMarketId)[0], systemProgram, ...arciumAccounts }`
   - On init_pool failure, retry up to 3 times (MAX_INIT_RETRIES = 3) with 2s exponential backoff before setting error state.

4. **create_market accounts and args:** The hook should accept an input object with `{ question: string, description: string, category: number, resolutionSource: string, resolutionTime: number, minBet: number }`. Derive market PDA, vault PDA, config PDA. Use `program.methods.createMarket(question, description, category, resolutionSource, new BN(resolutionTime), new BN(minBet))`.

5. **create_market accounts struct:** `{ creator: publicKey, config: configPda, creatorWhitelist: getCreatorWhitelistPda(publicKey)[0], market: getMarketPda(nextMarketId)[0], marketPool: getMarketPoolPda(nextMarketId)[0], marketVault: getVaultPda(nextMarketId)[0], usdcMint: USDC_MINT, tokenProgram: TOKEN_PROGRAM_ID, systemProgram: SystemProgram.programId }`. Add `getCreatorWhitelistPda` to the imports -- if it doesn't exist in pda.ts, derive it inline: `PublicKey.findProgramAddressSync([Buffer.from('whitelist'), publicKey.toBuffer()], PROGRAM_ID)`.

6. **onSuccess:** Invalidate `['markets']` query, show toast with market ID. Reset state after 1500ms delay.

7. **onError:** If init_pool failed (marketCreated is true), show retry toast. If create_market failed, show generic error toast.

8. **Return:** `{ ...mutation, step, retryCount, newMarketId, reset }`

**useMarketPool.ts** -- Create a hook that fetches MarketPool account data and subscribes to WebSocket changes for real-time pool initialization detection. Model on useMarket.ts pattern.

Key implementation details:

1. Use `useReadOnlyProgram()` so it works without wallet.
2. `useQuery` with key `['marketPool', marketId]`, fetching `program.account.marketPool.fetch(getMarketPoolPda(marketId)[0])`.
3. WebSocket subscription in useEffect: `connection.onAccountChange(poolPda, () => queryClient.invalidateQueries({ queryKey: ['marketPool', marketId] }))` with cleanup on unmount.
4. Export a helper function `isPoolInitialized(pool)`: checks if `yesPoolEncrypted` and `noPoolEncrypted` are NOT all zeros. Use `!pool.yesPoolEncrypted.every((b: number) => b === 0) || !pool.noPoolEncrypted.every((b: number) => b === 0)`.
5. Return the query result plus the `isPoolInitialized` computed boolean.

Modify the BetPlacement component to gate the bet form when the market pool is uninitialized.

1. **Add new PanelMode:** Add `'pool-initializing'` to the PanelMode union type (making it 13 modes total).

2. **Update getBetPanelMode signature and logic:**
   - Add `poolInitialized: boolean` as a 5th parameter to `getBetPanelMode`.
   - Add as the FIRST check in the function (before any other mode logic):
     ```
     if (market.state === 0 && !poolInitialized) return 'pool-initializing'
     ```
   - This means only active (state=0) markets with uninitialized pools show the gate. Resolved/disputed markets don't need it.

3. **Update BetPlacement component:**
   - Import `useMarketPool` and `isPoolInitialized` from `'#/hooks/useMarketPool'`.
   - Call `const poolQuery = useMarketPool(market.id)` at the top of BetPlacement.
   - Compute `const poolReady = poolQuery.data ? isPoolInitialized(poolQuery.data) : true` (default true to avoid flashing the gate on initial load before pool data arrives -- the on-chain init_pool may have already completed).
   - Pass `poolReady` as the 5th argument to `getBetPanelMode`.
   - Add `case 'pool-initializing':` to the switch statement, rendering `<PoolInitializingMode market={market} />`.

4. **Create PoolInitializingMode component:**
   - Section styled like other modes (rounded-xl bg-card p-6).
   - Shows "Market initializing..." label with spinner animation.
   - Shows explanatory text: "The encrypted pool is being initialized via MPC. This usually takes 10-30 seconds."
   - Track elapsed time since mount with useState + useEffect setInterval (increment every second).
   - After 30 seconds (`elapsed >= 30`), show a recovery button "Initialize Pool" styled with amber theme (matching dispute-escalate pattern).
   - The recovery button calls a simple init_pool transaction (same Arcium account derivation pattern). Use the same dynamic import and account derivation as in useCreateMarket Task 1. The recovery action can be a simple inline async function (no separate hook needed since it's a one-off recovery action).
   - While recovery TX is pending, show a spinner instead of the button.

5. **Important:** Do NOT change any existing mode logic. The pool-initializing check comes first but only triggers for state=0 + uninitialized pool. All other modes remain exactly as they are.
