  <files>app/src/components/search/SearchBar.tsx</files>
  <action>
Create a search bar component with as-you-type filtering and a dropdown showing matching markets.

**Implementation details:**

1. **Imports:**
   - `useState, useRef, useEffect, useCallback` from `react`
   - `Link, useNavigate` from `@tanstack/react-router`
   - `useMarkets` from `'#/hooks/useMarkets'`
   - `cn` from `'#/lib/utils'`
   - `CATEGORY_MAP, STATE_MAP` from `'#/lib/types'`
   - `Search, X` from `lucide-react` (search icon, clear button)

2. **State:**
   - `query: string` -- the search input value
   - `isOpen: boolean` -- whether the dropdown is visible
   - `selectedIndex: number` -- keyboard navigation index (-1 = none)

3. **Filtering logic:**
   - Get markets from `useMarkets()`.
   - When `query.length >= 1`, filter markets where any of these match (case-insensitive):
     a. `market.question.toLowerCase().includes(query.toLowerCase())`
     b. `market.resolutionSource.toLowerCase().includes(query.toLowerCase())`
     c. `CATEGORY_MAP[market.category]?.toLowerCase().includes(query.toLowerCase())`
   - Limit results to top 8 matches (Claude's discretion on max count).
   - Sort: live markets first (state 0), then by totalBets descending.

4. **Dropdown UI:**
   - Position: absolute, below the search input, full width of the search container, z-50.
   - Background: bg-card border border-border rounded-xl shadow-lg shadow-black/20 overflow-hidden
   - Each result item:
     - Clickable link to `/market/$id`
     - Market question (font-serif italic, line-clamp-1, text-sm)
     - Category badge (text-[10px] uppercase tracking-wider text-primary/50)
     - Status badge: color-coded by state:
       - Open (state 0): green dot + "Live"
       - Resolved/Finalized (state 2/4): muted "Resolved"/"Finalized"
       - Disputed (state 3): purple "In Dispute"
     - Hover state: bg-secondary/50
     - Active keyboard state: bg-secondary/70
   - "No results" message when query has text but no matches: "No markets found for 'query'"

5. **Keyboard navigation:**
   - ArrowDown: increment selectedIndex (wrap to 0 at end)
   - ArrowUp: decrement selectedIndex (wrap to end at -1)
   - Enter: navigate to selected result
   - Escape: close dropdown, blur input

6. **Click outside handling:**
   - Use a ref on the container div.
   - Add mousedown event listener on document.
   - If click target is outside the container ref, close dropdown.
   - Clean up listener on unmount.

7. **Input styling:**
   - Container: relative, w-full max-w-xs (desktop default).
   - Input: bg-secondary/50 border-0 rounded-lg px-3 py-1.5 pl-8 text-[13px] text-foreground placeholder:text-muted-foreground/50 focus:outline-none focus:ring-1 focus:ring-primary/20 transition-all
   - Search icon (lucide Search): absolute left-2.5, size-3.5, text-muted-foreground/50
   - Clear button (lucide X): absolute right-2, size-3.5, only visible when query is non-empty, cursor-pointer

8. **Open/close behavior:**
   - Open dropdown when input is focused AND query has text
   - Close on Escape, click outside, or clicking a result
   - Do NOT show dropdown when query is empty
