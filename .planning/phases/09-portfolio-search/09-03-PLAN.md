Implement a mobile-responsive header with hamburger menu and update SearchBar with a mobile overlay mode.

**Header changes:**

1. **State:** Add `const [menuOpen, setMenuOpen] = useState(false)`.

2. **Desktop layout (md+):** Preserve exactly as-is from Plan 02. The nav links, SearchBar, and WalletButton remain in the horizontal flex row. No changes to desktop.

3. **Mobile layout (below md):**
   - Replace the nav links + wallet button with two icon buttons:
     a. **Search icon button:** `<button>` with lucide `Search` icon (size-5). On click, open SearchBar in overlay mode (pass a prop or use a shared state).
     b. **Hamburger button:** `<button>` with lucide `Menu` icon (size-5) when closed, `X` icon when open.
   - Both buttons: min-h-[44px] min-w-[44px] (touch target compliance), flex items-center justify-center.
   - Use `hidden md:flex` on the desktop nav container, and `flex md:hidden` on the mobile icon buttons.

4. **Mobile menu panel:**
   - Conditionally rendered when `menuOpen` is true.
   - Position: fixed, top below header (top-[72px] approximately), left-0, right-0, z-40.
   - Background: bg-background/98 backdrop-blur-sm border-b border-border.
   - Content: vertical stack of links and wallet button:
     a. Markets link (Link to="/")
     b. Portfolio link (Link to="/portfolio")
     c. WalletButton component (wrapped in ClientOnly)
   - Each menu item: px-6 py-4 text-[15px], full width, min-h-[44px] for touch.
   - Clicking a link closes the menu (`setMenuOpen(false)`).
   - Close menu on route change: use `useEffect` that watches pathname and closes menu.

5. **Body scroll lock:** When menu is open, add `overflow-hidden` to body to prevent scroll-through. Use useEffect with `document.body.style.overflow`.

6. **Import additions:** `useState, useEffect` from react, `Menu, X, Search` from lucide-react.

**SearchBar changes for mobile overlay:**

1. **Add `mobile` prop:** `interface SearchBarProps { mobile?: boolean; onClose?: () => void }`.

2. **Mobile overlay mode (when `mobile` is true):**
   - Render as a full-width overlay: fixed top-0 left-0 right-0 z-[60] bg-background p-4.
   - Input takes full width, auto-focused.
   - Close button (X icon) on the right side of the input row.
   - Dropdown results render below, full-width.
   - Clicking a result calls `onClose?.()` to dismiss the overlay.
   - Pressing Escape calls `onClose?.()`.

3. **Desktop mode (default, `mobile` is false/undefined):**
   - Behavior unchanged from Plan 02.

4. **Header integration:**
   - Add state: `const [searchOpen, setSearchOpen] = useState(false)`.
   - Mobile search icon button sets `setSearchOpen(true)`.
   - Render `{searchOpen && <SearchBar mobile onClose={() => setSearchOpen(false)} />}` at the top level (outside the nav, as a portal-style overlay).
   - Desktop SearchBar remains as-is (non-mobile mode, inside the nav flex).

5. **Keyboard:** Mobile overlay has Escape listener to close.
