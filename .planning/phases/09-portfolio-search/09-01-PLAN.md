  <files>app/src/hooks/useUserPositions.ts</files>
  <action>
Create a new hook that fetches ALL UserPosition accounts for the connected wallet and enriches each position with its corresponding market data.

**Implementation details:**

1. **Imports:** useQuery from @tanstack/react-query, useWallet from @solana/wallet-adapter-react, useReadOnlyProgram from anchor.ts, mapPositionAccount and mapMarketAccount from types.ts, PROGRAM_ID from constants.ts.

2. **EnrichedPosition type:**
   ```typescript
   export interface EnrichedPosition {
     position: OnChainPosition
     market: OnChainMarket | null
   }
   ```

3. **Hook logic:** `useUserPositions()` -- no parameters needed.
   - Get `publicKey` from `useWallet()`
   - Get `program` from `useReadOnlyProgram()`
   - Use `useQuery` with queryKey `['positions', publicKey?.toBase58()]`
   - `enabled: !!publicKey`
   - `refetchInterval: 20_000` (match useMarkets polling cadence)
   - In queryFn:
     a. Fetch all UserPosition accounts filtered by user pubkey using Anchor's memcmp filter:
        ```typescript
        const positions = await program.account.userPosition.all([
          {
            memcmp: {
              offset: 8 + 8, // discriminator (8) + market_id (8), then user pubkey starts
              bytes: publicKey!.toBase58(),
            },
          },
        ])
        ```
        Map each to OnChainPosition via `mapPositionAccount(a.publicKey, a.account as any)`.
     b. Fetch all markets to enrich positions:
        ```typescript
        const markets = await program.account.market.all()
        const marketMap = new Map(
          markets.map(m => {
            const mapped = mapMarketAccount(m.publicKey, m.account as any)
            return [mapped.id, mapped]
          })
        )
        ```
     c. Join positions with markets:
        ```typescript
        return mappedPositions.map(pos => ({
          position: pos,
          market: marketMap.get(pos.marketId) ?? null,
        }))
        ```
   - Return the query result.

4. **Note on memcmp offset:** The UserPosition account layout in Anchor is: discriminator (8 bytes) + market_id (i64, 8 bytes) + user (Pubkey, 32 bytes). So the user field starts at offset 16. The memcmp filter at offset 16 with the user's base58 pubkey filters server-side on the RPC, returning only that user's positions.

5. **Fallback if memcmp offset is wrong:** If position accounts don't come back correctly, use a client-side fallback: fetch all UserPosition accounts with `program.account.userPosition.all()` then filter `.filter(p => p.user.equals(publicKey))`. This is less efficient but works as a safety net. Prefer the memcmp approach but include a comment noting the offset derivation.

   Create a function `categorizePositions(positions: EnrichedPosition[])` that returns:
   ```typescript
   {
     active: EnrichedPosition[]   // market state 0 (Open), 1 (Locked), 3 (Disputed)
     claimable: EnrichedPosition[] // market state 4 (Finalized), not claimed, user is winner
     history: EnrichedPosition[]   // claimed positions + losing positions in finalized markets
   }
   ```
   - **Active:** position where `market.state === 0 || market.state === 1 || market.state === 2 || market.state === 3` (any non-finalized market)
   - **Claimable:** `market.state === 4 && !position.claimed` AND user is on the winning side: `(market.winningOutcome === 1 && position.yesAmount > 0) || (market.winningOutcome === 2 && position.noAmount > 0)`
   - **History:** `market.state === 4 && (position.claimed || user is loser)`. A loser is: `(market.winningOutcome === 1 && position.noAmount > 0 && position.yesAmount === 0) || (market.winningOutcome === 2 && position.yesAmount > 0 && position.noAmount === 0)`. Also include any position where claimed=true.
   - If `market` is null (data not loaded), put in active as a safe default.

3. **Summary bar:**
   At the top of the portfolio, display a summary bar with 3 stats:
   - **Active Positions:** count of `active.length`
   - **Total Staked:** sum of `(position.yesAmount + position.noAmount) / 1_000_000` across all active positions, formatted as "XX.XX USDC"
   - **Winnings Claimed:** sum of claimed positions' estimated payouts (for claimed positions in finalized markets, approximate as the position amount -- exact payout requires on-chain calculation, so display the staked amount as a conservative lower bound). Format as "XX.XX USDC"
   - Layout: horizontal row of 3 stat cards, each with a label (text-[11px] uppercase tracking-wider text-muted-foreground) and value (font-mono text-lg). Style: rounded-xl bg-card p-4.

4. **Active section:**

     - Top row: category badge + market status/countdown
     - Market question (font-serif text-lg italic, line-clamp-2)
     - Position info: "Your bet: X.XX USDC on Yes/No" (font-mono text-sm)
     - Payout display: "Potential payout" with `<FogOverlay density="heavy"><span>Encrypted</span></FogOverlay>` (pools are hidden until resolution)
   - Determine user's side: if `position.yesAmount > 0` show "Yes", if `position.noAmount > 0` show "No". If both > 0 (rare edge case of betting both sides), show both.
   - Staked amount: `(position.yesAmount + position.noAmount) / 1_000_000` formatted to 2 decimal places.

5. **Claimable section:**
   - Section header: "Claimable" with count badge, styled with accent color to draw attention
   - Each position renders as a card with highlighted border (ring-1 ring-accent/20):
     - Market question (line-clamp-1, shorter since outcome is known)

     - Estimated payout: calculate `(userWinning * totalPool) / winningPool` then subtract 2% fee (200 bps). Show as "~XX.XX USDC" with the tilde indicating estimate.

   - **CRITICAL: Each claimable card needs its own useClaimPayout instance.** Create a `ClaimablePositionCard` sub-component that takes marketId and calls `useClaimPayout(marketId)` internally.
   - `totalPool = market.revealedYesPool + market.revealedNoPool`
   - `winningPool = market.winningOutcome === 1 ? market.revealedYesPool : market.revealedNoPool`
   - `userWinning = market.winningOutcome === 1 ? position.yesAmount : position.noAmount`

6. **History section:**
