**Step 3: Write hello-world circuit**
Add a `hello_world` circuit to `encrypted-ixs/src/lib.rs` alongside existing `init_pool`. This circuit should take two `Enc<Shared, u64>` inputs, add them together, and return `Enc<Mxe, u64>`. This validates:
- `Enc<Shared, T>` (user input) works
- `Enc<Mxe, T>` (persistent state) works
- Basic arithmetic in MPC works
- `to_arcis()` and `from_arcis()` lifecycle works

```rust
/// Hello-world circuit for environment validation.
/// Takes two encrypted u64 inputs (Shared) and returns their sum (Mxe-owned).
#[instruction]
pub fn hello_world(
    a_ctxt: Enc<Shared, u64>,
    b_ctxt: Enc<Shared, u64>,
) -> Enc<Mxe, u64> {
    let a = a_ctxt.to_arcis();
    let b = b_ctxt.to_arcis();
    let sum = a + b;
    Mxe::get().from_arcis(sum)
}
```

**Step 5: Write hello_world_callback instruction**
Following Pattern from research (Callback Writing State Back):
```rust
#[arcium_callback(encrypted_ix = "hello_world")]
pub fn hello_world_callback(
    ctx: Context<HelloWorldCallback>,
    output: SignedComputationOutputs<HelloWorldOutput>,
) -> Result<()> {
    let o = match output.verify_output(
        &ctx.accounts.cluster_account,
        &ctx.accounts.computation_account,
    ) {
        Ok(HelloWorldOutput { field_0 }) => field_0,
        Err(e) => {
            msg!("Computation failed: {}", e);
            return Err(ErrorCode::AbortedComputation.into());
        }
    };
    // For hello-world, just log the result. No state to update.
    msg!("Hello World MPC result: {:?}", o.ciphertexts);
    Ok(())
}
```
